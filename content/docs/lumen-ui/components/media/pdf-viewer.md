The `PdfViewer` component renders a card representing a PDF document. It displays the document title, a page count indicator, and a link to open or download the PDF. Use it for discharge summaries, clinical guidelines, insurance documents, or any PDF artifact generated by an agent.

## Props

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `url` | `string` | *required* | URL of the PDF document. |
| `title` | `string` | — | Display title for the document. If omitted, the file name from the URL is used. |
| `pages` | `number` | — | Total page count. Displayed as metadata when provided. |

## DSL Example

```
PdfViewer(
  url="/documents/discharge_summary.pdf",
  title="Discharge Summary",
  pages=3
)
```

## JSON Example

```json
{
  "type": "pdf_viewer",
  "props": {
    "url": "/documents/discharge_summary.pdf",
    "title": "Discharge Summary",
    "pages": 3
  }
}
```

## Composition

A PDF viewer alongside related context inside a tabbed layout:

```
Tabs(
  Card(title="Summary",
    Markdown(content="Patient discharged in stable condition. Follow up in 2 weeks with cardiology.")
  ),
  Card(title="Full Report",
    PdfViewer(
      url="/documents/discharge_summary.pdf",
      title="Discharge Summary — John Doe",
      pages=5
    )
  )
)
```

## Notes

- **Rendering**: The component displays a card-style preview with a PDF icon, title, and page count badge. Tapping the card opens the PDF URL in the platform's default viewer or browser tab.
- **No inline rendering**: PDFs are not rendered inline within the artifact. The component acts as an access point to the external document.
- **Page count**: The `pages` prop is informational only. It does not affect document loading or rendering behavior.
- **Generated artifacts**: When an agent generates a PDF artifact (e.g., a prior authorization letter), the server returns a `PdfViewer` component with the artifact's download URL pre-populated.
